The Core Principle Established This Session

Before this session, story voice had been applied selectively — usually caught when a draft landed wrong, then fixed. By the end of this session, the principle is documented, has a named failure mode, and has a named benchmark.

Story Voice Principle (now in CONTENT-RULES.md)

The opening text carries the day's theme via story. It never describes the artwork or artist. The image and music carry the encounter. The text carries the theme.

This same principle now extends to Context fields. Context drops the reader into a world. It does not summarize art history.

The Museum Label Failure Mode (named and documented)

When a Context field defaults to technique description, exhibition history, or movement names, it reads like a museum placard. The test: Read it aloud. Does it sound like a museum placard? Rewrite it.

The benchmark for every Context field: Myth Journey Day 1 (Eleusinian Mysteries). That text drops the reader into a living world. Every BTG Context was rewritten to that standard.

Drift Patterns: What Causes the Voice to Slip

Two patterns caused regressions this session. Both are now documented.

Drift Pattern 1: Content-First Writing

When content (artwork, artist, philosopher) is selected first, writing naturally starts by describing that content. The opening text ends up describing Dürer's technique or Plato's Symposium — correct information, wrong field.

The fix: Ask the theme question first. "What human experience does Day X carry?" Find the story that holds that experience. Then let the content items illuminate it.

Drift Pattern 2: Museum Label Default

Context fields default to art history because that information is easy to access and sounds authoritative. Technique, exhibition history, movement names — all present in training data, all wrong for KALLOS.

The fix: The museum label test. Read the field aloud. If it sounds like a placard, rewrite it. The question is not "what is true about this artwork?" but "what world did this artwork come from, and what was at stake there?"

Edit Quality KPIs

These metrics track whether content rules are reducing the editing burden over time. Three dimensions are measured each session: severity of edits made, the rule category that caused them, and who caught the issue. As rules mature, majors should decrease first, the Claude-caught ratio should rise, and repeat rule categories should disappear.

How to score a session

Edit severity: Major = full field rewritten. Minor = 1–2 sentences changed. Micro = word, phrase, or punctuation.

Reason categories: Museum label (Context field drift) | Content repeat (read-through test failure) | Voice drift (artwork description vs. theme) | Banned pattern (em dash, banned word, jargon) | Factual/accuracy | TTS/pacing | Process (unapproved write)

Who caught it: Claude = caught in autonomous audit before Sheri reviewed. Sheri = caught during Sheri's review pass.

Majors reduce first — structural rules (museum label, voice drift) landing means fewer full rewrites. Minors follow as the read-through test becomes habitual. Scoring note for session-close

At each session close, count every edit made to user-facing copy fields. Assign severity and reason. Note who caught it. Add a row to the table above. Do not count internal notes, Sanity entry notes, or verify flags — only fields that appear in the app.

If a field was revised more than once in a session, count each revision separately. A field that went through three drafts before landing is three edits, not one.
