Explore Cards — Build Brief

For: CC (developer)  |  Author: Sheri + Cowork  |  Date: June 9, 2026  |  Status: Ready to implement  |  Updated: June 9, 2026 (detail card flow confirmed, P&P link destination confirmed)

Problem / current production state

Explore surfaces currently pull data from the dailyPrompt schema, causing P&P-specific fields — context, P&P day title, actio — to bleed into the Explore UX. Explore is content-centric, not day-centric. All card data must come from contentItem directly.

Additional issues observed in the live app:

  • List card showing artworkHook excerpt (should show description)
  • Detail card showing dailyPrompt.context text (P&P field — must not appear on Explore)
  • Sacred art + landscape: currently tap straight into Visio Divina with no intermediate detail card. The new detail card described in this brief does not yet exist — it must be added.
  • All other content types (literature, food & wine, music, etc.) currently show "PRAY WITH THIS IMAGE" — this must be removed. These types never do Visio Divina. They get REFLECT only.
  • REFLECT expand has a border box — design decision was to remove it (text + chevron only)
Separate urgent fix (Task #113) — must ship before June 12:
getDailyPrompt() fallback returns the last seeded date (July 11) when today has no matching prompt. Fix: change fallback from order(date desc)[0] to date <= $today | order(date desc)[0]. This is independent of the Explore work and should be a fast patch.

Data source rule

All Explore card data comes from contentItem fields only.
No dailyPrompt fields on any Explore surface — not context, not actio, not dayTitle, not curatorNote. Use reverse references only to build contextual links (see below).

Contextual links

Below the action zone, separated by a divider. Shown only when the contentItem is referenced by a journey or a P&P day. These require reverse reference lookups (see GROQ below).

ConditionLink textDestination
contentItem is referenced by a journeyDay Part of: [Journey Name] → Journey overview page
contentItem is referenced by a dailyPrompt Pause & Ponder — [Month Day] → /library?view=all#[month-day] — always loads Library in "All P&P" view with date anchor, so the entry is visible regardless of the user's last Library filter state. Example: /library?view=all#june-12 scrolls to the June 12 entry. Anchor format: lowercase month + hyphen + day, no leading zero.
Multiple journeys or multiple P&P dates Expandable list (collapsed by default) Same destinations per item
No references Nothing shown —
Favorites / Saved do not affect P&P link destination. The link always goes to the dated P&P entry in the Library archive — not to the user's Saved section. Whether a user has the item favorited is irrelevant to this navigation. The P&P link's job is to show the item in its P&P context (day title, actio, etc.), which lives in the dated archive, not in the Saved view.

GROQ — reverse reference queries

Run these queries on the detail card load, keyed by contentItem._id.

Journey membership

*[_type == "journeyDay" && references($contentItemId)] {
  journey-> {
    _id,
    title,
    slug
  }
}

P&P day membership

*[_type == "dailyPrompt" && references($contentItemId)] {
  date,
  dayTitle
} | order(date asc)

Format the date as "Month Day" (e.g., "March 12") for the link label. Link destination = Library page.

Visio Divina — typography and color (all VD screens)

Font weight, colors, and opacity in Visio Divina currently look off. All VD screens (Gaze, Meditate, Pray, Contemplate, Action) and the Explore detail card must follow Journey step rules — with one specific difference for the type label color.

Typography — identical to Journey steps

  • Montserrat — labels and UI (e.g., "GAZE" step label)
  • Open Sans regular — body text
  • Open Sans italic — contemplative instructional text
  • Cormorant Garamond — quoted material only, 1.3rem minimum

All other text: standard cream on espresso, same weights and opacities as Journey steps. No custom opacity values, no one-off treatments.

The only colored text: the content item type label

The content item type label (e.g., "SACRED ART") is the only colored text on the detail card and in VD screens. Its color is the P&P gradient color for that content type — not the Explore theme color, not app green.

Examples: Sacred Art = Mineral Blue (#7ba2b8). Literature = Old Ochre (#c9a07c). Landscape = Sage Stone (#a8ae9a). See full type → color mapping in CLAUDE.md P&P Gradient Color System section.

Difference from Journeys: In Journey steps, the type identifier uses app green / sage (#4a7a62). On the Explore detail card and in VD, it uses the content type's P&P gradient color instead. Everything else is identical.

This is a polish pass, not a structural change. The goal is visual consistency across Journey steps and VD, with the type label as the single color signal for content type.

Confirmed flow summary (June 9, 2026)

"PRAY WITH THIS IMAGE" is retired everywhere. It no longer exists on any surface in Explore.

Sacred art + landscape: Currently tap straight into Visio Divina. New flow: Explore list → NEW detail card (image + type label + title + attribution + description + ENTER VISIO DIVINA →) → Gaze step. The detail card is a new screen to build.

All other types: Remove "PRAY WITH THIS IMAGE" wherever it appears. These types use REFLECT (text + chevron, no border box). They never enter Visio Divina.

Attribution format: Artist name + date/year reference (e.g., "Rembrandt van Rijn, c. 1669"). Source: contentItem.attribution.

P&P link destination: /library?view=all#[month-day]. Always loads Library in All view. Anchor format: lowercase, hyphenated, no leading zero (e.g., #june-12). Favorites/Saved status is irrelevant — link always goes to the dated archive entry, not the Saved section.
