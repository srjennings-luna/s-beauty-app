Task 1 — Zone 1 Nav Gap Fix

Affected files: app/explore/page.tsx · app/page.tsx · app/library/page.tsx

The Zone 1 nav bar sits too far down from the status bar on all three pages. The cause is a hardcoded pt-12 (48px) class on the Zone 1 container. This is too much padding on most devices and does not adapt to the actual iOS safe area inset.

Fix

On every Zone 1 container that currently has pt-12, remove that class and add the safe area top padding via inline style:

style={{ paddingTop: 'env(safe-area-inset-top, 16px)' }}

This adapts to the actual device — ~20px on older iPhones, ~59px on iPhone with Dynamic Island — so the nav sits correctly on all hardware.

Locations to fix

  • app/explore/page.tsx — two Zone 1 containers: line ~202 (theme detail state) and line ~296 (bubble landing state). Both have className="bg-parchment flex-shrink-0 pt-12".
  • app/page.tsx (Today tab) — check for same pt-12 pattern on the top-level header container. Apply same fix.
  • app/library/page.tsx — check for same pattern. Apply same fix.
Do not change any other spacing. Only the top padding on Zone 1 containers is affected.

Task 4 — Visual/List Icon Toggle on Explore

Affected file: app/explore/page.tsx

Add a view mode toggle to the Explore bubble landing page (not the detail screen — the landing screen with the bubbles). This allows users to switch between the physics bubble view and a simple list view of all six themes.

Toggle placement

Right-aligned in Zone 2 of the bubble landing header, on the same row as the existing "Explore" heading or just below it. Do not center it. Match the right-alignment pattern of the ALL/SAVED toggle on the Library page.

Toggle design — icon only, no text labels

Two icon buttons side by side. Active state: highlighted background behind the active icon (same pattern as Library's ALL/SAVED toggle). Inactive state: no background.

Circles icon (Visual/Bubbles mode):

  • Three overlapping circles, espresso #16110d, varying opacity to show depth
  • Back circle: fill-opacity="0.3"
  • Middle circle: fill-opacity="0.55"
  • Front circle: fill-opacity="1"
  • Semi-stacked arrangement (not perfectly aligned — offset so all three are visible)

List icon (List mode):

  • Three horizontal rectangles, espresso #16110d, solid fill
  • No rounded corners — rx="0" or no rx attribute. Sharp ends only. This is non-negotiable per design system.
No rounded corners on any element of this toggle. The design system rule is absolute: border-radius: 0 everywhere. This includes the list icon lines, the toggle container, and the active state background. Do not add rx values or border-radius to any part of this component.

List view — theme rows

When the user selects the list icon, replace the bubble canvas with a simple list of all six themes. Each row:

  • Fresco palette color dot (filled circle, ~10px) on the left — color comes from theme.color field in Sanity
  • Theme name (Montserrat, same weight as other labels)
  • Chevron right on the far right
  • No secondary tagline text
  • Tapping a row navigates to the theme detail screen (same as tapping a bubble)
  • Rows flush edge-to-edge with a hairline divider between them — rgba(22,17,13,0.12)

The list view uses the same parchment background and Zone 1/Zone 2 header as the bubble landing. Only the content area changes.

State persistence

The selected view mode (bubbles or list) should persist within the session using React state. No need for localStorage persistence — reverting to bubbles on next visit is fine.

Task 7 — Bubble Entrance Animation

Affected file: components/ThemeBubbleCanvas.tsx (or equivalent)

The bubbles currently appear too fast when the Explore page loads. This feels jarring for a meditative app. The entrance should feel like objects gently settling into place, not launching onto the screen.

Target feel

Slow, calm, staggered. Each bubble should drift in gently — either fading in with low initial velocity, or starting from a resting position and slowly coming to life. The entrance should take 1.5–2.5 seconds total, with each bubble slightly offset in timing so they don't all appear at once.

Guidance

  • Reduce initial velocity to near zero — bubbles should barely be moving when they first appear
  • Stagger the fade-in or entry of each bubble by 150–250ms
  • Do not use a dramatic scale-up or bounce entrance — subtle opacity fade is preferred
  • Respect prefers-reduced-motion — if set, skip the entrance animation and show bubbles immediately at rest

Task 8 — Bubble Bounce Physics

Affected file: components/ThemeBubbleCanvas.tsx (or equivalent)

When bubbles hit the canvas walls or are released after being dragged, the bounce is too energetic for a meditative app. The physics should feel like objects drifting in water — slow, absorbing, calming — not like rubber balls.

Two specific adjustments

Wall bounce (coefficient of restitution): Reduce significantly. Current behavior has bubbles bouncing off walls with most of their energy retained. Target: bubbles should absorb the wall, losing 80–90% of velocity on impact. A coefficient around 0.1–0.15 is the target range.

Velocity damping after user release: Increase the damping factor so bubbles decelerate quickly after the user lets go. They should slow to near-stillness within 1–2 seconds of release, not continue drifting for a long time.

These changes should feel like a significant behavioral shift on device — noticeably calmer. If the result still feels energetic, reduce further. The bar is: a person in a contemplative state should not feel interrupted by the physics.
Do not touch bubble sizes. The current sizes are not the final spec — a breathing animation (grow/shrink between 90px–110px diameter) is planned for a future session. Leave bubble sizing logic untouched.
