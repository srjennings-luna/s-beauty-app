Task 1 — Sanity Schema: Journey Type Fields

Add three new fields to the journey schema in Sanity. These are purely additive — no migration required. Existing journeys will have empty values and default to the "standard" section.

journeyTypestring — dropdown
Controls which section this journey appears in on the Journeys page. Options: standard (default, goes in "Journeys" section), intro (goes in "Where to Begin" section, gets Start Here badge), companion (goes in "Episode Companions" section, gets companion card treatment). Make this a Sanity list with these three options. Default: standard.
showNamestring — optional
For companion journeys only. The name of the show or series this journey pairs with. Example: "Seeking Beauty". Appears in the teal strip at the top of companion cards. Leave blank for non-companion journeys.
episodeLabelstring — optional
For companion journeys only. The episode identifier shown on the companion card strip. Example: "Episode 1". Appears on the right side of the companion card teal strip alongside showName. Leave blank for non-companion journeys.

Deploy Sanity Studio: cd sanity && npx sanity deploy. Confirm the three new fields appear in Studio on the journey document type before proceeding to Task 2.

Task 2 — Journeys Page UI Redesign

The current Journeys page (/journeys) shows a flat list of all journeys. Redesign it to show three labeled sections based on the journeyType field. All cards remain visible — no collapse, no "show more".

Section structure

Section 1 — Where to Begin
Journeys where journeyType === 'intro'. Section label: "WHERE TO BEGIN" in gold small caps. Cards in this section display a "START HERE" badge overlaid on the top-right corner of the artwork image area. Badge styling: small gold pill, text "START HERE", positioned absolute top-right over the hero image.
Section 2 — Journeys
Journeys where journeyType === 'standard' OR journeyType is null/undefined (default fallback). Section label: "JOURNEYS" in gold small caps. Standard card treatment — no badge, no special styling. This is the existing card design.
Section 3 — Episode Companions
Journeys where journeyType === 'companion'. Section label: "EPISODE COMPANIONS" in gold small caps. Companion card treatment — see full spec below. Only render this section if at least one companion journey exists.

Section dividers

Separate sections with a thin horizontal rule (1px solid, muted espresso tone, e.g. rgba(42,32,21,0.4)) with the section label immediately below the rule. Match the existing KALLOS label style used elsewhere in the app (small caps, gold, Montserrat or equivalent).

Companion card design spec

Companion card — full spec

The companion card shares the same base structure as the standard card (artwork image, title, day count, description) but adds the following:

  • Show strip at top: A slim horizontal band above the artwork image area. Background: dark teal (#0d1a12). Left side: showName in small caps, teal-light color (#5fa87a), ~9px, letter-spacing 0.16em. Right side: episodeLabel in same small caps, slightly more muted teal (#4a7a62), ~10px. Strip height approximately 32px with 7px vertical padding.
  • Card border: Replace the standard card border with a teal-tinted border: 1px solid #1e3028.
  • Tag color: The "COMPANION" tag uses teal background (rgba(74,122,98,0.2)) and teal text (#5fa87a) instead of the standard gold. The day count tag remains gold as usual.
  • Watch cue: Below the description text, add a small italic line: "Watch the episode before beginning" in a muted teal tone (#4a7a62), approximately 11px, using the serif font (Cormorant Garamond or equivalent). This is a static string — not from Sanity.
  • Artwork area: Retain the artwork image at the same height as standard cards. The show strip sits above it, not over it.

GROQ query update

Update the GROQ query that fetches journeys for the Journeys page to include the three new fields:

Fields to add to the journey list query
Add journeyType, showName, and episodeLabel to whatever GROQ projection currently fetches journey list data. These should be included alongside title, slug, description, totalDays, and the hero image.

TypeScript

Update the Journey type in lib/types.ts to include the three new optional fields: journeyType?: 'standard' | 'intro' | 'companion', showName?: string, episodeLabel?: string.

Rendering logic

On the Journeys page, split the fetched journeys array into three groups based on journeyType. Render each group under its labeled section. Group order: intro first, then standard, then companion. If a group is empty, skip the section header and render nothing for that section. Do not render an empty "Episode Companions" section before the companion journey is entered in Sanity.

Do not hardcode journey slugs. The section assignment must come from the journeyType field in Sanity — not from a list of known slugs in code. Sheri will set journeyType in Studio for each journey.
