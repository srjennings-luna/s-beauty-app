KALLOS

Phase 1: Foundation & Seeding — Implementation Spec
Handoff document for Claude Code • February 2026
What this document is: A complete spec for Phase 1 implementation. It tells Claude Code exactly what to build, what to change, and what to leave alone. Each section ends with a handoff summary — the specific files to create or modify, in what order.

Overview: What Changes

The app is transitioning from an EWTN companion app (episode-based, location-first) to KALLOS — an independent contemplative discovery app (content-first, theme-organized). Phase 1 lays the foundation.

Schema Status

RETIRE: episode MODIFY: artwork → contentItem KEEP: splashPage KEEP: traditionReflection (expand themes) NEW: theme NEW: journey NEW: dailyPrompt

Migration note: The existing Sanity dataset has live content (episodes, artworks, tradition reflections, splash pages). Don't delete the episode schema immediately — deprecate it. Existing artwork documents can be migrated to the new contentItem schema by adding the new fields; the existing field structure is mostly compatible.

1. New Schema: theme

Document

theme

The organizing principle for all content. Themes tag every content item and structure every Journey. The five founding themes are: Light, Silence, Suffering & Beauty, Creation, Home / The Restless Heart.

FieldTypeRequiredDescription
title string Yes Theme name. e.g., "Light"
slug slug Yes URL-safe version. Source: title. e.g., "light"
question string Yes The central question. e.g., "What does light reveal that darkness hides?"
description text (3 rows) Yes 2–3 sentence description of the theme for the Explore/Journey UI
color string No Hex color for UI pills and theme cards. e.g., "#C19B5F"
order number No Display order in theme lists. Initial: 1–5.
image image (hotspot) No Optional hero image for theme cards in Explore UI

2. Modified Schema: artwork → contentItem

Why one unified type: All 7 content types share core fields (title, image, themes, reflection questions). Journeys, the Explore tab, and the Daily Prompt all need to reference content — one type means one query, one reference type, one filter system. Sanity's conditional field visibility keeps the Studio UX clean for each content type.
Document

contentItem

Replaces the current artwork schema. The contentType field replaces locationType and expands from 5 location categories to 7 content types.

Shared Fields (always visible)

FieldTypeRequiredDescription
contentType string (radio) Yes Options: sacred-art, thinker, literature, music, food-wine, landscape, watch-listen
Initial: sacred-art. Replaces locationType.
title string Yes Content title. e.g., "Pietà" or "Into Great Silence"
image image (hotspot) Yes High-res image. Min 1200px, up to 4500px for detail works. Every content item must have an image — this is a core KALLOS principle.
description text (3 rows) Yes Short description for cards and previews. 1–2 sentences.
context text (6 rows) No Deeper context. Replaces historicalSummary. Why this matters, what to notice, how it connects to the tradition.
themes array of references → theme Yes (min 1) Which themes this content speaks to. e.g., a Caravaggio could be tagged Light + Suffering & Beauty.
reflectionQuestions array of string No 1–3 questions for contemplation. Used in Reflect step of Journeys and on content detail page.
curatorNote text (3 rows) No Why this was chosen. The human voice behind the curation. Shown on detail view.

Location Fields (visible for all types — optional for non-place content)

FieldTypeRequiredDescription
locationName string No e.g., "St. Peter's Basilica", "Grande Chartreuse"
city string No e.g., "Rome", "Florence"
country string No Default: empty (not Italy anymore — KALLOS is global)
coordinates object { lat: number, lng: number } No For Explore map view. Only populate when content has a meaningful physical location.

3. New Schema: journey

Document

journey

A 7-day curated exploration of one theme across multiple content types. The integrated curriculum model — all content answers the same central question from different angles.

FieldTypeRequiredDescription
title string Yes Journey title. e.g., "Light"
slug slug Yes URL-safe. Source: title.
theme reference → theme Yes Which theme this Journey explores
description text (4 rows) Yes What the seeker will explore over 7 days. Shown on Journey card.
heroImage image (hotspot) Yes Cover image for Journey card. High-res.
estimatedMinutesPerDay number No Default: 10. Shown as "~10 min/day"
isPublished boolean No Default: false. Toggle when all 7 days are ready.
order number No Display order in Journey list.
days array of journeyDay objects Yes Exactly 7 days. See journeyDay structure below.

journeyDay (inline object within days array)

Each day follows the 5-step structure: Open → Encounter → Reflect → Connect → Go Deeper.

FieldTypeRequiredDescription
dayNumber number Yes 1–7
dayTitle string Yes e.g., "The First Light" or "What Darkness Teaches"
openImage image (hotspot) Yes ALWAYS required. The visual anchor for the day. Min 1200px. This is the image the user sees first and returns to. Every day has an image regardless of content type.
openText text (3 rows) No Brief opening text (~1 min read). Sets the day's frame. Optional if image alone is sufficient.
encounterContent reference → contentItem Yes The primary content for this day. Links to a contentItem of any type. This is the 3–5 min encounter.
encounterGuidance text (3 rows) No Optional editorial guidance for how to engage with this content. e.g., "Listen with your eyes closed for the first minute."
reflectQuestions array of string Yes (1–2) Journey-specific reflection questions for this day (may differ from the contentItem's own reflection questions).
connectThread string No One sentence threading to tomorrow. e.g., "Tomorrow: what happens when the light goes away?"
goDeeper array of references → traditionReflection No Optional tradition reflections for this day. User's choice to explore.

4. New Schema: dailyPrompt

Document

dailyPrompt

The Today tab centerpiece. One curated piece of content + one reflection question, editorially chosen. Not algorithmic. Designed to be opened every morning.

FieldTypeRequiredDescription
date date Yes The date this prompt appears. One per day. Format: YYYY-MM-DD.
content reference → contentItem Yes The featured content for this day.
promptQuestion text (2 rows) Yes The single reflection question for the day. Curated specifically for this pairing.
curatorNote text (3 rows) No Optional: why this was chosen for today. The human editorial voice.
theme reference → theme No Optional theme tag. Helps if user is on a Journey — they see the connection.

5. Modified Schema: traditionReflection

Modify existing

traditionReflection

Keep everything. Add one field: themes (array of references → theme). This replaces the current string-based theme dropdown so tradition reflections connect to the same theme system as all content.

ChangeDetails
Add field: themes Array of references → theme. Replaces the current string dropdown. Keep the old theme field temporarily for migration, but hide it in the Studio.
Keep everything else authorType, title, summary, shortQuote, source, order, era — all unchanged.

Do these in order — each step depends on the one before it.

Create theme.ts, contentItem.ts, journey.ts, dailyPrompt.ts in sanity/schemaTypes/. Update index.ts to export all schemas. Keep episode.ts but add deprecation comment. Keep splashPage.ts unchanged. Update traditionReflection.ts with themes array.

Update lib/types.ts with new types (ContentItem, Theme, Journey, JourneyDay, DailyPrompt). Update lib/sanity.ts with new GROQ queries. Remove episode-specific queries.

Files: lib/types.ts (rewrite), lib/sanity.ts (rewrite queries)

Step 3: EWTN Removal + Rebranding

Update layout.tsx metadata, remove all EWTN references, fix tailwind.config colors, rename localStorage keys. Update splash pages for KALLOS branding (content comes from Sanity — just update the schema references if needed).

Files: app/layout.tsx, tailwind.config.ts, hooks/useFavorites.ts, lib/favorites.ts, app/splash/page.tsx

Step 4: Navigation + Page Structure

Update Navigation.tsx from 3 tabs to 4 tabs (Today, Explore, Journeys, Library). Create new route folders. Rename app/map/ → build app/explore/. Create app/journeys/ and app/library/. Rebuild app/page.tsx as Today tab.

Files: components/ui/Navigation.tsx (update), app/page.tsx (rebuild as Today), app/explore/page.tsx (new, based on map page), app/journeys/page.tsx (new), app/journeys/[slug]/page.tsx (new), app/library/page.tsx (new, replaces favorites)

Step 5: Bug Fixes + Cleanup

Fix all known bugs from RECOMMENDATIONS.md. Delete data/episodes.ts. Remove unused code. Add error/empty states. Fix accessibility issues. Audit globals.css for rounded corners.

Files: app/globals.css (audit), data/episodes.ts (delete), various components for aria-labels and error states

Step 6: Test & Deploy

Run npm run build to catch TypeScript errors. Test all routes. Verify Visio Divina still works. Check map markers. Commit and push to GitHub → Vercel auto-deploys.

Commands: npm run build, npm run dev (manual testing), git add/commit/push

10. Design System Reminders

These are non-negotiable. Read CLAUDE_REFERENCE.md before writing any UI code.

RuleDetails
No rounded cornersborder-radius: 0 everywhere. Sharp/square corners. Classical aesthetic.
Dark themeBackground: #203545 (dark teal). Gradient: lighter teal top → darker teal bottom.
Gold accent#C19B5F — for CTAs, active states, gold highlights. NOT #C9A227.
TypographyMontserrat (headlines), Open Sans (body), Cormorant Garamond (scripture/quotes).
White opacity levels100% (titles), 80% (body), 50% (secondary), 40% (tertiary).
ImagesMin 1200px wide. Up to 4500px. 8x zoom built in. Pinch-to-zoom via react-zoom-pan-pinch.
Safe areasPadding for phone notches and home indicators.
